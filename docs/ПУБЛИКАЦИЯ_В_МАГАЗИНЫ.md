# Публикация DogPaw в App Store и Google Play

Чтобы приложение можно было скачать напрямую из **App Store** (iOS) и **Google Play** (Android), нужно собрать нативные сборки и отправить их в магазины. Для проектов на **Expo** это делается через **EAS (Expo Application Services)**.

---

## Что понадобится

| Магазин      | Что нужно |
|-------------|-----------|
| **App Store** | Аккаунт [Apple Developer Program](https://developer.apple.com/programs/) (платно, ~99 USD/год), доступ к [App Store Connect](https://appstoreconnect.apple.com). |
| **Google Play** | Аккаунт [Google Play Console](https://play.google.com/console) (разовый взнос ~25 USD). |

Плюс аккаунт **Expo** (бесплатный): [expo.dev](https://expo.dev), через него работают EAS Build и EAS Submit.

---

## Шаг 1. Установка EAS CLI

```bash
npm install -g eas-cli
```

Войдите в аккаунт Expo:

```bash
eas login
```

(Если аккаунта нет — создайте на [expo.dev](https://expo.dev).)

---

## Шаг 2. Подключение проекта к EAS

В корне проекта выполните:

```bash
eas build:configure
```

Будет создан/обновлён файл `eas.json` (он уже есть в репозитории). При необходимости отредактируйте профили сборки в `eas.json`.

---

## Шаг 3. Сборка приложения

### Сборка для Google Play (Android)

```bash
eas build --platform android --profile production
```

Сборка идёт в облаке Expo. По окончании будет ссылка на **AAB** (Android App Bundle) — этот файл загружается в Google Play Console.

### Сборка для App Store (iOS)

```bash
eas build --platform ios --profile production
```

Нужна подписка Apple Developer. В процессе EAS может запросить Apple ID и пароль (или использовать сохранённые учётные данные). На выходе — файл **.ipa** для загрузки в App Store Connect.

### Сборка для обеих платформ

```bash
eas build --platform all --profile production
```

---

## Шаг 4. Отправка в магазины (EAS Submit)

После успешной сборки можно отправить билд в магазин одной командой.

### Android (Google Play)

1. В [Google Play Console](https://play.google.com/console) создайте приложение и заполните карточку (описание, скриншоты, политика конфиденциальности и т.д.).
2. Создайте [сервисный аккаунт](https://developers.google.com/android-publisher/get-start#creating_a_service_account) с доступом к Play Console, скачайте JSON-ключ и положите его в корень проекта как `google-service-account.json` (или укажите другой путь в `eas.json` → `submit.production.android.serviceAccountKeyPath`).
3. В `eas.json` в секции `submit.production.android` укажите путь к ключу и при необходимости `track` (например, `internal`, `alpha`, `production`).
4. Выполните:

```bash
eas submit --platform android --profile production
```

EAS предложит выбрать последний билд или указать путь к AAB.

### iOS (App Store)

1. В [App Store Connect](https://appstoreconnect.apple.com) создайте приложение (Bundle ID должен совпадать с `ios.bundleIdentifier` в `app.json` — сейчас `com.dogowner.app`).
2. В `eas.json` в секции `submit.production.ios` укажите свой `appleId`, `ascAppId` (ID приложения в App Store Connect) и `appleTeamId` (Team ID в Apple Developer).
3. Выполните:

```bash
eas submit --platform ios --profile production
```

При необходимости EAS запросит Apple ID и пароль (или app-specific password).

## Важные моменты

- **Backend API:** В приложении в `src/api/config.ts` для продакшена используется `https://api.dogpaw.app`. Перед публикацией нужно развернуть бэкенд по этому адресу (или изменить URL в конфиге и пересобрать приложение).
- **Иконка и сплэш:** Проверьте, что в `assets/` есть `icon.png`, `adaptive-icon.png`, `splash.png` и что они соответствуют [требованиям Expo](https://docs.expo.dev/develop/user-interface/splash-screen-and-app-icon/).
- **Версия:** В `app.json` указаны `version` (1.0.0) и `ios.buildNumber` / `android.versionCode` при необходимости. В `eas.json` для профиля `production` включён `autoIncrement` — EAS может автоматически увеличивать номер сборки.
- **Политика конфиденциальности:** Для Google Play и часто для App Store нужна публичная ссылка на политику конфиденциальности (если собираете данные пользователей).

---

## Альтернатива: скачать билд и загрузить вручную

Если не хотите использовать EAS Submit:

1. После `eas build` откройте ссылку из вывода команды или зайдите в [expo.dev](https://expo.dev) → ваш проект → Builds.
2. Скачайте **AAB** (Android) или **.ipa** (iOS).
3. **Android:** в Google Play Console → Release → Create new release → загрузите AAB.
4. **iOS:** в App Store Connect создайте версию приложения, затем в Xcode или через Transporter загрузите .ipa.
