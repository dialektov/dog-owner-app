Дата: 2026-03-31

## Среда

- Backend: локально, `go run .`, порт `8080`
- Frontend: Expo Metro, порт `8082`
- API base URL: `EXPO_PUBLIC_API_BASE_URL=http://192.168.1.54:8080`

1. Остановлены дублирующиеся процессы backend/Expo.

3. Expo поднят на фиксированном порту `8082`.

Результат:

- PASS: backend отвечает `{"status":"ok"}`

## Этап 5. Owner-only admin сценарий

Проверка:

- попытка `register` для `idialektov@gmail.com` -> `email already exists`;
- попытка `login` с тестовым паролем -> `invalid credentials`.

Результат:

- BLOCKED (нужен действующий пароль владельца для полного e2e-теста выдачи admin-прав).
- При этом owner-only guard и запрет для не-владельцев подтверждены.

1. Лента:
   - пост с фото;
   - лайк;
   - комментарий;
   - 4 режима сортировки.
2. Друзья:
   - добавить по email;

3. Умная карта:
   - фильтр 5 категорий в легенде;

4. База знаний:
   - пользователь отправляет статью;
   - админ одобряет/отклоняет;
   - удаление статьи админом.
5. Админ:
   - владелец выдает права другу (требует вход владельца).

Статус:

- READY FOR MANUAL QA.

## Этап 7. Дополнительный ретест после пользовательского фидбека (карта/цвета/модерация)

- Для энциклопедии:
  - добавлен отдельный блок `Ожидают модерации` для админа;
  - `GET /articles` переведен на optional-auth middleware, чтобы админ-статус учитывался по токену.
